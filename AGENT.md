# **AGENT.md**

### **กฎการทำงานสำหรับ AI Agent (Cursor) — โปรเจกต์เฝ้าท่อ (FaoTor)**

อ่านไฟล์นี้ก่อนเริ่มเขียนโค้ดทุกครั้ง เพื่อให้เข้าใจ Context และกฎของโปรเจกต์นี้

---

## **1\. โปรเจกต์นี้คืออะไร (สรุปสั้น)**

ระบบให้ชุมชนถ่ายรูปท่อระบายน้ำที่เสี่ยงอุดตัน ส่งเข้าเว็บแอป แล้ว AI (Gemini Vision API) วิเคราะห์ระดับความเสี่ยง เจ้าหน้าที่ดู Dashboard จัดลำดับความสำคัญในการลอกท่อ

รายละเอียดเต็มอยู่ที่ FaoTor\_SRS.md — **ให้อ่านไฟล์นั้นก่อนเสมอถ้าไม่แน่ใจเรื่อง Requirement**

## **2\. Scope ที่ทำอยู่ตอนนี้ (สำคัญที่สุด)**

กำลังทำ **Phase 1 (MVP)** เท่านั้น ตาม FR-101 ถึง FR-106 ใน SRS

**ห้ามทำสิ่งต่อไปนี้ในตอนนี้** (เป็นของ Phase 2 ยังไม่ต้องทำ):

* ห้ามสร้างระบบ Login/Authentication ที่ซับซ้อน  
* ห้ามทำแผนที่ (Map View) — Phase 1 ใช้ตาราง/List ธรรมดาพอ  
* ห้ามทำระบบ Gamification/แต้มสะสม  
* ห้ามเชื่อม API พยากรณ์ฝน — Phase 1 มีแค่ AI วิเคราะห์ภาพอย่างเดียว

ถ้า Agent เห็นว่างานที่กำลังทำเข้าเงื่อนไขด้านบน ให้หยุดถามก่อน ไม่ต้องทำเองแล้วมาบอกทีหลัง

## **3\. เทคโนโลยีที่ต้องใช้ (ดูรายละเอียดที่ TECH\_STACK.md)**

* ห้ามเพิ่ม Library หรือ Service ใหม่ที่ไม่ได้ระบุใน TECH\_STACK.md โดยไม่ถามก่อน  
* **ห้ามใช้ Service ที่มีค่าใช้จ่าย** — งบประมาณโปรเจกต์คือ 0 บาท ต้องใช้ Free Tier เท่านั้น (เช่น Gemini API ต้องใช้ Free Tier ไม่ใช่ Paid Tier)

## **4\. โครงสร้างไฟล์ (ดูรายละเอียดที่ FOLDER\_STRUCTURE.md)**

* วางไฟล์ตามโครงสร้างที่กำหนดไว้เท่านั้น ห้ามสร้างโฟลเดอร์ใหม่เองโดยไม่จำเป็น  
* ถ้าจำเป็นต้องสร้างโฟลเดอร์ใหม่ ให้บอกเหตุผลก่อน

## **5\. หลักการเขียนโค้ด**

* โค้ดต้อง**เรียบง่ายที่สุดที่ทำงานได้** (Simplicity over cleverness) — ทีมนี้คือนักเรียนม.ปลายที่ Vibe Coding ไม่ใช่ทีม Dev มืออาชีพ โค้ดต้องอ่านง่าย แก้ง่าย  
* Comment ในโค้ดเป็นภาษาไทยหรืออังกฤษก็ได้ แต่ต้องอธิบายว่า "ทำไม" ไม่ใช่แค่ "ทำอะไร"  
* ทุกฟีเจอร์ต้องทดสอบได้จริงผ่านหน้าเว็บ (Manual test ผ่านเบราว์เซอร์) ไม่ใช่แค่ Code สวยแต่รันไม่ได้จริง  
* ถ้า Error หรือไม่แน่ใจว่าทำถูกไหม ให้หยุดและถามก่อน ดีกว่าทำต่อแบบเดาไปเรื่อย

## **6\. ลำดับความสำคัญเมื่อเวลาน้อย (Timeline บีบมาก)**

ถ้าเวลาไม่พอ ให้เรียงความสำคัญดังนี้ (ทำสิ่งบนสุดให้ได้ก่อน):

1. อัปโหลดรูป \+ เรียก Gemini Vision API ได้จริง (Core ที่สุด — ถ้าไม่มีนี่ \= ไม่มีโปรเจกต์)  
2. บันทึกผลลง Database  
3. หน้า Dashboard แสดงรายการ  
4. ปุ่มเปลี่ยนสถานะ  
5. ความสวยงามของ UI (สำคัญน้อยสุด ทำหลังสุด)

## **7\. เมื่อไม่แน่ใจ**

ให้ Agent ถามผู้ใช้ก่อนเสมอในกรณี:

* จะเพิ่ม Library ใหม่ที่ไม่ได้ระบุไว้  
* จะเปลี่ยนโครงสร้างฐานข้อมูลที่ต่างจาก SRS  
* Requirement ใน Prompt ขัดกับ SRS หรือ AGENT.md นี้
